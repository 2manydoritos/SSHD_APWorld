# Skyward Sword HD Archipelago Setup Guide

## Overview

This guide will walk you through setting up and playing **Skyward Sword HD** in an Archipelago multiworld.

## Requirements

### Minimum System Requirements
- **Nintendo Switch** (for playing the game)
- **Ryujinx Emulator** (on PC, for Archipelago client)
- **Python 3.10 or higher** (for generating multiworlds)
- **Skyward Sword HD** ROM file (legally obtained)

### Software Setup
- Ryujinx emulator with Archipelago plugin support
- Archipelago client (Python-based)
- SSHD APWorld package

## Installation

### Step 1: Install Archipelago

1. Download and install Archipelago from [archipelago.gg](https://archipelago.gg)
2. Follow the standard Archipelago installation procedure for your operating system
3. Verify installation by running the Launcher

### Step 2: Install SSHD APWorld

1. Download the latest `sshd-apworld.zip` from the [GitHub Releases](https://github.com/your-repo/SSHD-APWorld/releases)
2. Extract the ZIP file to your Archipelago installation folder (typically `%LOCALAPPDATA%\Archipelago\` on Windows)
3. The folder structure should look like:
   ```
   Archipelago/
   ├── worlds/
   │   └── sshd/
   │       ├── __init__.py
   │       ├── Items.py
   │       ├── Locations.py
   │       └── ...
   └── ...
   ```

### Step 3: Set Up Ryujinx with Archipelago Plugin

1. **Install Ryujinx**: Download from [Ryujinx GitHub](https://github.com/Ryujinx/Ryujinx)
2. **Load Skyward Sword HD**: Open your Skyward Sword HD ROM in Ryujinx
3. **Install Archipelago Plugin**:
   - Locate your Ryujinx folder
   - Navigate to `Mods` directory
   - Download the Skyward Sword HD Archipelago mod from [mod-source]
   - Extract into the Mods folder
   - The plugin will inject the Archipelago connection code into the game

### Step 4: Generate Your Multiworld

1. Open **Archipelago Launcher** and select "Generate"
2. Create a new generation directory or use an existing one
3. Add a YAML settings file for Skyward Sword HD:
   - See [Options Guide](#options-overview) below for available settings
   - Example YAML:
     ```yaml
     name: YourPlayerName
     game: Skyward Sword HD
     Progression Dungeons:
       Skyview: true
       Earth Temple: true
       Lanayru Mining Facility: true
       Ancient Cistern: true
       Sandship: true
       Volcano Summit: true
       Sky Keep: true
     ```
4. Click "Generate" and wait for the multiworld creation
5. The output will be a `.zip` file with your generated ROM and settings

### Step 5: Extract and Play

1. Download your generated multiworld ROM from the Archipelago server (or receive the local file)
2. Extract the ROM file to a location accessible by Ryujinx
3. Load the ROM in Ryujinx
4. The game will automatically connect to the Archipelago server
5. Start playing!

## Running the Archipelago Client

The Archipelago client automatically connects when you load the game in Ryujinx, but you can also:

1. Open **Archipelago Launcher**
2. Select "Skyward Sword HD Client" from the list
3. Enter your connection details (server address, player name, password if required)
4. The client will remain open during your play session to track items and locations

## Configuration and Options

The SSHD Archipelago world includes many customizable options:

### Progression Options
- **Progression Dungeons**: Choose which dungeons are progression (required to complete the game)
- **Progression Bosses**: Control boss difficulty and progression requirements
- **Required Dungeon Count**: Minimum dungeons to complete the game (default: 4)

### Randomization Options
- **Randomize Entrances**: Shuffle dungeon/cave/building entrances
- **Randomize Treasures**: Shuffle treasure chest locations
- **Triforce Shuffle**: Control where Triforce Shards are placed

### Quality of Life
- **Key Drop**: Make small keys available from any source (default: true)
- **Music Randomization**: Shuffle in-game music
- **Convenience Shop**: Access to items without finding them

### Game Path Configuration
- **Extract Path**: Path to extracted Skyward Sword HD files (auto-detected by default)
- **Custom Options**: JSON-based advanced configuration

For a complete list of options, check the YAML template generated by Archipelago.

## Troubleshooting

### Game won't connect to Archipelago
- **Check server connectivity**: Ensure your Archipelago server is running and reachable
- **Check player name**: Verify you're using the correct player name from your YAML
- **Check client logs**: Look for error messages in the Archipelago client output
- **Restart the client**: Close and reopen the Archipelago client

### Items not appearing in-game
- **Check connection status**: Verify the client shows "Connected" to the server
- **Check world state**: Open the Spoiler Log to verify items are assigned to locations
- **Restart the game**: Try reloading the game in Ryujinx

### ROM generation fails
- **Check YAML syntax**: Use the YAML validator in Archipelago Launcher
- **Verify game path**: Ensure the Extract Path points to valid Skyward Sword files
- **Check disk space**: Ensure you have enough space for ROM generation
- **Review error log**: Detailed errors are shown in the generation output

### Ryujinx emulation issues
- **Update Ryujinx**: Ensure you have the latest version
- **Check GPU drivers**: Update your graphics drivers
- **Adjust settings**: Try changing Ryujinx graphics backend (OpenGL vs Vulkan)
- **Check mod conflicts**: Disable other mods and test with only the Archipelago mod

## Advanced Setup

### Custom Logic Rules
Edit `Rules.py` in the SSHD world package to customize item placement logic.

### Multiple Playthroughs
Generate multiple YAML files with different settings for variety:
- Different dungeon counts
- Different entrance randomization options
- Different item pool configurations

### Server Hosting
To host your own Archipelago server:
1. Run `MultiServer.py` from the Archipelago folder
2. Load your generated multiworld ZIP file
3. Share the server address with other players

## Tips for Playing

1. **Check hints frequently**: Open the hint interface to learn where important items are
2. **Use the spoiler log**: If playing solo, refer to the generation's spoiler log for item locations
3. **Map all locations**: The tracker will help you keep track of which locations you've checked
4. **Plan your route**: With randomized dungeons, plan which dungeons to do in which order
5. **Communicate**: In multiplayer sessions, share information about item locations and trades
